
========== business-processes.docx ==========
Описание бизнес-процессов для сервиса организации закупок
Сервис организации закупок включает несколько ключевых бизнес-процессов, которые охватывают взаимодействие между покупателями, организаторами и поставщиками. Ниже приведено детальное описание этих процессов.
1. Процесс регистрации и авторизации
1.1. Регистрация пользователя
Участники: Покупатель, Организатор, Поставщик.
Шаги:
Пользователь заполняет форму регистрации (email, пароль, роль).
Система проверяет уникальность email.
Система создает учетную запись пользователя.
Пользователь получает подтверждение о успешной регистрации.
Результат: Пользователь зарегистрирован в системе.
1.2. Авторизация пользователя
Участники: Покупатель, Организатор, Поставщик.
Шаги:
Пользователь вводит email и пароль.
Система проверяет данные и генерирует JWT-токен.
Пользователь получает токен для доступа к системе.
Результат: Пользователь авторизован и имеет доступ к личному кабинету.
2. Процесс создания и управления закупками
2.1. Создание закупки
Участники: Организатор.
Шаги:
Организатор заполняет форму создания закупки (название товара, единица измерения, город, описание, комиссия).
Система создает закупку и добавляет ее в горизонтальный слайдер.
Закупка отображается в общем чате.
Результат: Закупка создана и доступна для участия.
2.2. Участие в закупке
Участники: Покупатель.
Шаги:
Покупатель находит закупку в слайдере и нажимает на нее.
Покупатель заполняет форму добавления в закупку (количество товара, город получения).
Система добавляет покупателя в закупку.
Результат: Покупатель становится участником закупки.
2.3. Голосование за поставщика
Участники: Покупатели, Организатор.
Шаги:
Организатор запускает голосование за поставщика.
Покупатели голосуют за одного из предложенных поставщиков.
Система подсчитывает голоса и определяет победителя.
Результат: Поставщик утвержден для закупки.
2.4. Завершение закупки
Участники: Организатор, Покупатели, Поставщик.
Шаги:
Организатор нажимает кнопку "Стоп-сумма".
Покупатели подтверждают участие в закупке.
Организатор утверждает поставщика.
Система формирует сводную таблицу чеков и отправляет ее поставщику.
Организатор закрывает закупку.
Результат: Закупка завершена, поставщик получает данные для отгрузки.
3. Процесс управления балансом
3.1. Пополнение баланса
Участники: Покупатель.
Шаги:
Покупатель заполняет форму пополнения баланса (сумма, способ оплаты).
Система перенаправляет покупателя на страницу оплаты через API банка.
После успешной оплаты баланс покупателя пополняется.
Результат: Баланс покупателя увеличен.
3.2. Вывод средств
Участники: Покупатель.
Шаги:
Покупатель заполняет форму вывода средств (сумма, реквизиты счета).
Система обрабатывает запрос и переводит средства на указанный счет.
Результат: Средства списаны с баланса покупателя.
4. Процесс управления новостями
4.1. Публикация новости
Участники: Организатор, Поставщик.
Шаги:
Организатор или поставщик заполняет форму создания новости (заголовок, содержание).
Система публикует новость в ленте новостей.
Результат: Новость доступна для просмотра всем пользователям.
4.2. Редактирование новости
Участники: Организатор, Поставщик.
Шаги:
Организатор или поставщик выбирает новость для редактирования.
Система открывает форму редактирования.
После внесения изменений новость обновляется.
Результат: Новость обновлена.
5. Процесс управления данными поставщика
5.1. Создание карты компании
Участники: Поставщик.
Шаги:
Поставщик заполняет форму создания карты компании (название, адрес, ИНН и т.д.).
Система создает карту компании и публикует ее в публичной части кабинета.
Результат: Карта компании доступна для просмотра.
5.2. Загрузка прайс-листа
Участники: Поставщик.
Шаги:
Поставщик загружает файл с прайс-листом.
Система обрабатывает файл и публикует прайс-лист.
Результат: Прайс-лист доступен для просмотра.
6. Процесс управления чатом
6.1. Отправка сообщения
Участники: Покупатель, Организатор, Поставщик.
Шаги:
Пользователь вводит текст сообщения и нажимает "Отправить".
Система добавляет сообщение в чат.
Результат: Сообщение отображается в чате.
6.2. Редактирование сообщения
Участники: Покупатель, Организатор, Поставщик.
Шаги:
Пользователь выбирает сообщение для редактирования.
Система открывает форму редактирования.
После внесения изменений сообщение обновляется.
Результат: Сообщение обновлено.
7. Процесс управления отгрузками
7.1. Формирование отгрузки
Участники: Поставщик.
Шаги:
Поставщик получает сводную таблицу чеков от организатора.
Поставщик формирует отгрузку на основе данных из таблицы.
Результат: Отгрузка сформирована.
7.2. Отправка закрывающих документов
Участники: Поставщик.
Шаги:
Поставщик загружает закрывающие документы.
Система отправляет документы покупателям.
Результат: Покупатели получают закрывающие документы.
8. Процесс управления историей закупок
8.1. Просмотр истории закупок
Участники: Покупатель, Организатор.
Шаги:
Пользователь переходит в раздел "История закупок".
Система отображает список завершенных закупок.
Результат: Пользователь видит историю закупок.
9. Процесс управления приглашениями
9.1. Приглашение поставщика
Участники: Организатор.
Шаги:
Организатор заполняет форму приглашения поставщика (email, текст приглашения).
Система отправляет приглашение поставщику.
Результат: Поставщик получает приглашение.
9.2. Приглашение покупателя
Участники: Организатор.
Шаги:
Организатор заполняет форму приглашения покупателя (текст приглашения, ссылка на закупку).
Система отправляет приглашение покупателю.
Результат: Покупатель получает приглашение.
10. Процесс управления голосованиями
10.1. Создание голосования
Участники: Покупатель, Организатор.
Шаги:
Пользователь заполняет форму создания голосования (вопрос, варианты ответа).
Система создает голосование и добавляет его в чат.
Результат: Голосование доступно для участия.
10.2. Участие в голосовании
Участники: Покупатели.
Шаги:
Покупатель выбирает вариант ответа в голосовании.
Система учитывает голос и обновляет результаты.
Результат: Результаты голосования обновлены.
Если у вас есть дополнительные пожелания или уточнения, пожалуйста, сообщите!

========== buttons.docx ==========
Расширенное описание функционала кнопок для сервиса организации закупок
Ниже приведено детальное описание функционала всех кнопок, которые будут использоваться в сервисе. Описание разделено по ролям пользователей: покупатель, организатор и поставщик.
1. Функционал кнопок для покупателя
1.1. Личный кабинет покупателя
Создать запрос:
Описание: Позволяет покупателю создать новый запрос на товар.
Функционал:
Открывает форму с полями: название товара, количество, город получения, примечание.
После заполнения формы запрос добавляется в общий чат.
Результат: Запрос отображается в вертикальной ленте чата.
Поиск товаров:
Описание: Позволяет покупателю искать товары по названию.
Функционал:
Открывает строку поиска.
При вводе названия товара и нажатии "Искать" происходит переход в чат, где упоминаются соответствующие товары.
Результат: Отображение результатов поиска в чате.
Мои запросы:
Описание: Отображает список запросов, созданных покупателем.
Функционал:
Показывает запросы в виде списка с количеством новых заявок по каждому товару.
При нажатии на запрос происходит переход в чат, где упоминаются соответствующие закупки.
Результат: Переход в чат с закупками по выбранному товару.
Мои закупки:
Описание: Отображает список закупок, в которых участвует покупатель.
Функционал:
Показывает закупки в виде списка с указанием активности (например, необходимость подтверждения участия).
При нажатии на закупку происходит переход в чат с деталями закупки.
Кнопка "Удалить из заявки" позволяет выйти из закупки.
Результат: Переход в чат закупки или удаление из заявки.
Мои приглашения и сообщения:
Описание: Отображает приглашения в закупки и сообщения от других пользователей.
Функционал:
Показывает список приглашений и сообщений.
При нажатии на приглашение происходит переход в чат закупки.
При нажатии на сообщение открывается окно для ответа.
Результат: Переход в чат или ответ на сообщение.
История закупок:
Описание: Отображает историю закупок, в которых участвовал покупатель.
Функционал:
Показывает список завершенных закупок с указанием общей суммы и статуса.
Результат: Просмотр истории закупок.
Пополнить баланс / Вывести средства:
Описание: Управление балансом покупателя.
Функционал:
"Пополнить баланс" — открывает страницу оплаты через API банка.
"Вывести средства" — открывает форму для вывода средств на указанный счет.
Результат: Изменение баланса покупателя.
2. Функционал кнопок для организатора
2.1. Личный кабинет организатора
Создать закупку:
Описание: Позволяет организатору создать новую закупку.
Функционал:
Открывает форму с полями: название товара, единица измерения, город получения, описание, размер комиссии (1-4%).
После заполнения формы закупка добавляется в горизонтальный слайдер.
Результат: Закупка отображается в слайдере и чате.
Открытые закупки:
Описание: Отображает список активных закупок, созданных организатором.
Функционал:
Показывает закупки в виде списка.
При нажатии на закупку происходит переход в чат с деталями закупки.
Результат: Переход в чат закупки.
Закупки в стадии оплаты:
Описание: Отображает закупки, которые находятся в стадии оплаты.
Функционал:
Показывает список закупок с указанием даты остановки закупки.
При нажатии на закупку происходит переход в закрытый чат участников.
Результат: Переход в закрытый чат.
История закупок:
Описание: Отображает историю завершенных закупок.
Функционал:
Показывает список закупок с указанием общей суммы и вознаграждения организатора.
Результат: Просмотр истории закупок.
Создать новость:
Описание: Позволяет организатору публиковать новости.
Функционал:
Открывает форму для создания новости (заголовок, содержание).
После публикации новость отображается в ленте новостей.
Результат: Новость публикуется в ленте.
2.2. Кнопки в слайдере (видимые только организатору)
Стоп-сумма:
Описание: Останавливает закупку и создает закрытый чат для участников.
Функционал:
После нажатия участники получают уведомление с просьбой подтвердить участие.
Участники, подтвердившие участие, переносятся в закрытый чат.
Результат: Создание закрытого чата.
Утвердить поставщика:
Описание: Утверждает выбранного поставщика.
Функционал:
После нажатия поставщик добавляется в информацию о закупке.
Результат: Поставщик утвержден.
Создать и отправить таблицу чеков:
Описание: Формирует сводную таблицу с чеками покупателей.
Функционал:
После нажатия создается таблица с данными о платежах.
Таблица отправляется поставщику.
Результат: Поставщик получает таблицу.
Закрыть закупку:
Описание: Завершает закупку.
Функционал:
После нажатия закупка перемещается в историю.
Результат: Закупка завершена.
3. Функционал кнопок для поставщика
3.1. Личный кабинет поставщика
Создать карту компании:
Описание: Позволяет поставщику создать карту компании.
Функционал:
Открывает форму с полями: название компании, адрес, ИНН, контактный телефон, электронная почта.
Результат: Карта компании создана.
Загрузить прайс-лист:
Описание: Позволяет поставщику загрузить прайс-лист.
Функционал:
Открывает форму для загрузки файла.
После загрузки прайс-лист отображается в публичной части кабинета.
Результат: Прайс-лист загружен.
Таблица заказов:
Описание: Отображает сводные таблицы с заказами.
Функционал:
Показывает таблицы с данными о заказах.
Кнопка "Отправить закрывающие документы" позволяет отправить документы покупателям.
Результат: Документы отправлены.
История отгрузок:
Описание: Отображает историю отгрузок.
Функционал:
Показывает список завершенных отгрузок.
Результат: Просмотр истории.
Написать на страницу новостей:
Описание: Позволяет поставщику публиковать новости.
Функционал:
Открывает форму для создания новости.
Результат: Новость опубликована.

========== forms-extended.docx ==========
Расширенное описание функционала форм для сервиса организации закупок
Формы в сервисе используются для ввода данных, создания запросов, управления закупками, публикации новостей и других задач. Ниже приведено детальное описание функционала всех форм, разделенных по ролям пользователей: покупатель, организатор и поставщик.
1. Формы для покупателя
1.1. Форма создания запроса
Назначение: Позволяет покупателю создать запрос на товар.
Поля:
Название товара (обязательное): Название товара, который ищет покупатель.
Количество товара (обязательное): Количество товара, которое необходимо.
Город получения товара (обязательное): Город, где покупатель хочет получить товар.
Примечание (необязательное): Дополнительная информация о запросе (например, предпочтения по бренду или срокам).
Функционал:
После заполнения формы запрос добавляется в общий чат.
Покупатель может редактировать или удалить запрос через личный кабинет.
Результат: Запрос отображается в вертикальной ленте чата.
1.2. Форма добавления в закупку
Назначение: Позволяет покупателю добавиться в закупку.
Поля:
Название товара (автозаполнение): Название товара из закупки.
Количество товара (обязательное): Количество товара с учетом единиц измерения, заданных организатором.
Город получения товара (обязательное): Город, где покупатель хочет получить товар.
Функционал:
После заполнения формы покупатель добавляется в закупку.
Покупатель может редактировать данные до подтверждения закупки организатором.
Результат: Покупатель становится участником закупки.
1.3. Форма пополнения баланса
Назначение: Позволяет покупателю пополнить баланс.
Поля:
Сумма (обязательное): Сумма для пополнения.
Способ оплаты (обязательное): Выбор способа оплаты (банковская карта, электронный кошелек и т.д.).
Функционал:
После заполнения формы происходит перенаправление на страницу оплаты через API банка.
Результат: Баланс покупателя пополнен.
1.4. Форма вывода средств
Назначение: Позволяет покупателю вывести средства с баланса.
Поля:
Сумма (обязательное): Сумма для вывода.
Реквизиты счета (обязательное): Реквизиты для перевода средств.
Функционал:
После заполнения формы средства выводятся на указанный счет.
Результат: Средства списаны с баланса покупателя.
2. Формы для организатора
2.1. Форма создания закупки
Назначение: Позволяет организатору создать новую закупку.
Поля:
Название товара (обязательное): Название товара для закупки.
Единица измерения (обязательное): Единица измерения товара (штуки, килограммы, литры и т.д.).
Город получения товара (обязательное): Город, где будет получен товар.
Описание (необязательное): Дополнительная информация о закупке.
Размер комиссии (обязательное): Комиссия организатора (от 1% до 4%).
Минимальное количество товара (необязательное): Минимальное количество товара для запуска закупки.
Функционал:

========== service-vision.docx ==========
Видение сервиса организации закупок
Сервис представляет собой платформу для взаимодействия между покупателями, организаторами и поставщиками, где основным инструментом является чат с фильтрацией и вертикально-горизонтальной навигацией. Сервис объединяет в себе функционал для создания закупок, управления заявками, общения между участниками и автоматизации процессов, связанных с закупками.
1. Основные цели сервиса
Упрощение процесса закупок: Сервис позволяет покупателям находить нужные товары, а организаторам — эффективно управлять закупками.
Автоматизация рутинных процессов: Формирование сводных таблиц, голосование за поставщиков, управление платежами и отгрузками автоматизированы.
Прозрачность и удобство: Все участники закупки имеют доступ к актуальной информации о товарах, поставщиках, платежах и сроках.
Гибкость и адаптивность: Сервис адаптируется под потребности пользователей, предоставляя разные сценарии взаимодействия в зависимости от роли (покупатель, организатор, поставщик).
2. Ключевые особенности сервиса
2.1. Чат с фильтрацией
Вертикальная лента: Отображает запросы покупателей и сообщения.
Горизонтальный слайдер: Показывает закупки, созданные организаторами.
Фильтрация: Пользователи могут фильтровать информацию по товарам, закупкам и участникам.
Интерактивность: Возможность переходить к закупкам, отправлять сообщения, приглашать участников и редактировать сообщения.
2.2. Личные кабинеты
Покупатель:
Создание запросов на товары.
Управление закупками и запросами.
Просмотр истории закупок и баланса.
Организатор:
Создание и управление закупками.
Утверждение поставщиков.
Формирование сводных таблиц и управление платежами.
Поставщик:
Создание карты компании и загрузка прайс-листов.
Управление заказами и отгрузками.
Публикация новостей и акций.
2.3. Управление закупками
Создание закупки: Организатор указывает товар, единицы измерения, город получения и другие параметры.
Голосование за поставщика: Покупатели голосуют за поставщиков, а организатор утверждает победителя.
Стоп-сумма: Организатор останавливает закупку, создавая закрытый чат для участников.
Формирование таблиц: Автоматическое создание сводных таблиц с данными о платежах и отгрузках.
2.4. Новости и общение
Лента новостей: Отображает новости от организаторов и поставщиков.
Чат: Позволяет участникам общаться, задавать вопросы и обсуждать детали закупок.
Голосования: Пользователи могут создавать голосования для принятия коллективных решений.
3. Пользовательские сценарии
3.1. Сценарий покупателя
Покупатель создает запрос на товар через личный кабинет.
Находит закупку в горизонтальном слайдере и добавляется в нее.
Участвует в голосовании за поставщика.
После утверждения поставщика подтверждает участие в закупке и переходит в закрытый чат.
Получает товар и оплачивает его через сервис.
3.2. Сценарий организатора
Организатор создает закупку, указывая товар, единицы измерения и город получения.
Управляет закупкой: запускает голосование, утверждает поставщика, останавливает закупку.
Формирует сводные таблицы с данными о платежах и отправляет их поставщику.
Закрывает закупку после завершения всех процессов.
3.3. Сценарий поставщика
Поставщик регистрируется, заполняет карту компании и загружает прайс-лист.
Получает приглашения в закупки и участвует в них.
Общается с покупателями и организаторами в чате.
Получает сводные таблицы с заказами и отправляет закрывающие документы.
4. Преимущества сервиса
4.1. Для покупателей
Удобный поиск товаров и закупок.
Прозрачность процесса закупок.
Возможность участвовать в голосованиях и принимать коллективные решения.
4.2. Для организаторов
Автоматизация рутинных процессов (формирование таблиц, управление платежами).
Удобное управление закупками через чат и личный кабинет.
Возможность привлекать поставщиков и покупателей через приглашения.
4.3. Для поставщиков
Публикация прайс-листов и новостей.
Участие в закупках и общение с покупателями.
Получение сводных данных о заказах и отгрузках.
5. Технологическая основа
Frontend: Адаптивный интерфейс с вертикальной и горизонтальной навигацией.
Backend: NestJS для обработки запросов, управления данными и интеграции с платежными системами.
База данных: PostgreSQL для хранения данных о пользователях, закупках, платежах и отгрузках.
Безопасность: JWT для аутентификации, шифрование данных и защита от несанкционированного доступа.
6. Дальнейшее развитие
Интеграция с платежными системами: Поддержка различных способов оплаты (банковские карты, электронные кошельки).
Мобильное приложение: Разработка мобильной версии сервиса для удобства пользователей.
Расширение функционала: Добавление новых ролей (например, логистические компании) и возможностей (например, управление доставкой).
7. Заключение
Сервис организации закупок представляет собой универсальную платформу, которая объединяет покупателей, организаторов и поставщиков, обеспечивая прозрачность, удобство и автоматизацию процессов. Его гибкость и адаптивность позволяют учитывать потребности всех участников, делая процесс закупок максимально эффективным и комфортным.
