Расширенное описание функционала форм для сервиса организации закупокФормы в сервисе используются для ввода данных, создания запросов, управления закупками, публикации новостей и других задач. Ниже приведено детальное описание функционала всех форм, разделенных по ролям пользователей: покупатель, организатор и поставщик.1. Формы для покупателя1.1. Форма создания запросаНазначение: Позволяет покупателю создать запрос на товар.Поля:Название товара (обязательное): Название товара, который ищет покупатель.Количество товара (обязательное): Количество товара, которое необходимо.Город получения товара (обязательное): Город, где покупатель хочет получить товар.Примечание (необязательное): Дополнительная информация о запросе (например, предпочтения по бренду или срокам).Функционал:После заполнения формы запрос добавляется в общий чат.Покупатель может редактировать или удалить запрос через личный кабинет.Результат: Запрос отображается в вертикальной ленте чата.1.2. Форма добавления в закупкуНазначение: Позволяет покупателю добавиться в закупку.Поля:Название товара (автозаполнение): Название товара из закупки.Количество товара (обязательное): Количество товара с учетом единиц измерения, заданных организатором.Город получения товара (обязательное): Город, где покупатель хочет получить товар.Функционал:После заполнения формы покупатель добавляется в закупку.Покупатель может редактировать данные до подтверждения закупки организатором.Результат: Покупатель становится участником закупки.1.3. Форма пополнения балансаНазначение: Позволяет покупателю пополнить баланс.Поля:Сумма (обязательное): Сумма для пополнения.Способ оплаты (обязательное): Выбор способа оплаты (банковская карта, электронный кошелек и т.д.).Функционал:После заполнения формы происходит перенаправление на страницу оплаты через API банка.Результат: Баланс покупателя пополнен.1.4. Форма вывода средствНазначение: Позволяет покупателю вывести средства с баланса.Поля:Сумма (обязательное): Сумма для вывода.Реквизиты счета (обязательное): Реквизиты для перевода средств.Функционал:После заполнения формы средства выводятся на указанный счет.Результат: Средства списаны с баланса покупателя.2. Формы для организатора2.1. Форма создания закупкиНазначение: Позволяет организатору создать новую закупку.Поля:Название товара (обязательное): Название товара для закупки.Единица измерения (обязательное): Единица измерения товара (штуки, килограммы, литры и т.д.).Город получения товара (обязательное): Город, где будет получен товар.Описание (необязательное): Дополнительная информация о закупке.Размер комиссии (обязательное): Комиссия организатора (от 1% до 4%).Минимальное количество товара (необязательное): Минимальное количество товара для запуска закупки.Функционал:После заполнения формы закупка добавляется в горизонтальный слайдер.Организатор может редактировать закупку до ее завершения.Результат: Закупка отображается в слайдере и чате.2.2. Форма создания новостиНазначение: Позволяет организатору публиковать новости.Поля:Заголовок (обязательное): Заголовок новости.Содержание (обязательное): Текст новости.Функционал:После заполнения формы новость публикуется в ленте новостей.Организатор может редактировать или удалить новость.Результат: Новость отображается в ленте новостей.2.3. Форма отправки таблицы чековНазначение: Позволяет организатору отправить сводную таблицу чеков поставщику.Поля:Данные о покупателях (автозаполнение): ФИО, адрес доставки, сумма заказа.Общая сумма закупки (автозаполнение): Общая сумма закупки.Функционал:После заполнения формы таблица отправляется поставщику.Результат: Поставщик получает таблицу с данными о заказе.3. Формы для поставщика3.1. Форма создания карты компанииНазначение: Позволяет поставщику создать карту компании.Поля:Название компании (обязательное): Полное наименование компании.Юридический адрес (обязательное): Юридический адрес компании.Почтовый адрес (обязательное): Почтовый адрес компании.Фактический адрес (обязательное): Фактический адрес компании.ОКВЭД (обязательное): Код ОКВЭД.ОГРН (обязательное): Основной государственный регистрационный номер.ИНН (обязательное): Идентификационный номер налогоплательщика.Контактный телефон (обязательное): Контактный телефон компании.Электронная почта (обязательное): Электронная почта компании.Функционал:После заполнения формы карта компании публикуется в публичной части кабинета.Поставщик может редактировать данные.Результат: Карта компании отображается в публичной части.3.2. Форма загрузки прайс-листаНазначение: Позволяет поставщику загрузить прайс-лист.Поля:Файл прайс-листа (обязательное): Загрузка файла с прайс-листом.Популярные товары (необязательное): Список из 20 самых популярных товаров с ценами.Функционал:После загрузки файла прайс-лист отображается в публичной части кабинета.Поставщик может обновить прайс-лист.Результат: Прайс-лист доступен для просмотра.3.3. Форма отправки закрывающих документовНазначение: Позволяет поставщику отправить закрывающие документы покупателям.Поля:Документы (обязательное): Загрузка файлов с документами.Комментарий (необязательное): Дополнительная информация для покупателей.Функционал:После заполнения формы документы отправляются покупателям.Результат: Покупатели получают закрывающие документы.4. Общие формы4.1. Форма авторизацииНазначение: Позволяет пользователям войти в систему.Поля:Электронная почта (обязательное): Электронная почта пользователя.Пароль (обязательное): Пароль пользователя.Функционал:После заполнения формы пользователь авторизуется в системе.Результат: Пользователь получает доступ к личному кабинету.4.2. Форма регистрацииНазначение: Позволяет новым пользователям зарегистрироваться в системе.Поля:Электронная почта (обязательное): Электронная почта пользователя.Пароль (обязательное): Пароль пользователя.Роль (обязательное): Выбор роли (покупатель, организатор, поставщик).Функционал:После заполнения формы пользователь регистрируется в системе.Результат: Пользователь получает доступ к личному кабинету.5. Технические требования к формамВалидация: Все поля форм должны быть валидированы на стороне клиента и сервера.Адаптивность: Формы должны корректно отображаться на устройствах с разными разрешениями экрана.Уведомления: После успешного заполнения формы пользователь должен получать уведомление (например, "Запрос успешно создан").Обработка ошибок: В случае ошибки (например, неправильный формат email) пользователь должен получить соответствующее сообщение.
